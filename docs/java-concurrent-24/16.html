    16 | Semaphore：如何快速实现一个限流器

    lorraine 2022-07-05

    Semaphore，称为“信号量”，由迪杰斯特拉（Dijkstra）于 1965 年提出

    # 信号量模型

    信号量模型主要由 1 个计数器、1 个等待队列、3 个方法组成。这 3 个方法分别为init()、up()、down()，外面只能通过这 3 种方法来访问计数器和等待队列。方法的详细使用如下：

    • init()：设置计数器的初始值
    • up()：计数器的值 + 1。如果此时计数器的值 <= 0，那么唤醒等待队列中的一个线程，并从等待队列移除
    • down()：计数器的值 - 1。如果此时计数器的值 < 0，那么将当前线程阻塞并加入等待队列，否则继续执行

    其中init()、up()、down()这 3 个方法都是原子性的，即线程安全的

    # 在 Java 中如何使用信号量

    信号量模型的down()、up()最早对应的是操作系统的 PV 操作，也称为 PV 原语 在 Java 中，信号量是由java.util.concurrent.Semaphore实现的，信号量模型的down()、up()对应了Semaphore类中的acquire()和release()，Semaphore类能够保证这 3 个方法都是原子操作 使用Semaphore实现计数器（计数器：同一时刻只能有一个线程操作）

    static int count;
//初始值设为 1，即同一时刻只允许一个线程访问
static final Semaphore s = new Semaphore(1);
//用信号量保证互斥
static void addOne() {
	s.acquire();
	try {
		count += 1;
	} finally {
		s.release();
	}
}

    Semaphore比Lock多的一个功能是：Semaphore能够允许多个线程访问一个临界区

    # 用Semaphore实现一个限流器

    什么时候会需要多个线程访问一个临界区 ？ 比如工作中遇到的各种池化资源，例如连接池、对象池、线程池等等。最常见的是数据库连接池，在同一时刻，一定是允许多个线程同时使用连接池的，当然，每个连接在被释放前，是不允许其他线程使用的 例子：设计一个对象池的限流器，对象池指的是一次性创建出 N 个对象，之后所有的线程重复利用这 N 个对象，当然对象在被释放前，也是不允许其他线程使用的

    class ObjPool<T, R> {
	final List<T> pool;
	// 用信号量实现限流器
	final Semaphore sem;
	// 构造函数
	ObjPool(int size, T t){
		pool = new Vector<T>(){};
		for(int i=0; i<size; i++){
			pool.add(t);
		}
		sem = new Semaphore(size);
	}
	// 利用对象池的对象，调用 func
	R exec(Function<T,R> func) {
		T t = null;
		sem.acquire();
		try {
			t = pool.remove(0);
			return func.apply(t);
		} finally {
			pool.add(t);
			sem.release();
		}
	}
}

// 创建对象池
ObjPool<Long, String> pool = new ObjPool<Long, String>(10, 2);
// 通过对象池获取 t，之后执行
pool.exec(t -> {
	System.out.println(t);
	return t.toString();
});

    具体做法：用一个 List 来保存对象实例，用Semaphore实现限流器。关键的代码是ObjPool里面的exec()方法，这个方法里面实现了限流的功能。在这个方法里面，步骤如下：

    1. 首先调用acquire()（与之匹配的是在 finally 里面调用 release()），假设对象池的大小是 10，信号量的计数器初始化为 10，那么前 10 个线程调用acquire()方法，都能继续执行，相当于通过了信号灯，而其他线程则会阻塞在acquire()方法上
    2. 对于通过信号灯的线程
      1. 我们为每个线程分配了一个对象 t（这个分配工作是通过pool.remove(0)实现的）
      2. 分配完之后会执行一个回调函数func，而函数的参数正是前面分配的对象 t
      3. 执行完回调函数之后，它们就会释放对象（这个释放工作是通过pool.add(t)实现的），同时调用release()方法来更新信号量的计数器
      4. 如果此时信号量里计数器的值 <= 0，那么说明有线程在等待，此时会自动唤醒等待的线程
