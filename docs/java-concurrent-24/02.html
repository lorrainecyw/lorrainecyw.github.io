    02 | Java 内存模型：解决可见性和有序性问题

    lorraine 2022-07-01

    本讲重点：

    • volatile 关键字的作用
    • happens-before 规则

    由上一讲 (opens new window)可知，可见性问题是缓存导致的，有序性问题是编译优化导致的，那么合理解决可见性和有序性问题地方案应当是按需禁用缓存和编译优化（完全禁用的话会严重影响程序的性能） Java 内存模型（JMM，Java Memory Model）规范了 JVM 如何提供按需禁用缓存和编译优化的方法，这些方法包括了synchronized、volatile等，以及Happens-Before规则

    # volatile 的作用

    1. 当一个变量被volatile修饰后，相当于告诉 JVM，这个变量是共享且不稳定的，对这个变量的读写不能使用 CPU 缓存，需要到内存中去进行读取或写入，这样就保证了可见性
    2. volatile还能够禁止指令重排序

    # Happens-Before 规则（8 项）

    Happens-Before要表达的是：前面一个操作的结果对后面的操作是可见的。Happens-Before约束了编译器的优化行为，虽然允许编译优化，但是要求编译优化后必须遵守Happens-Before规则

    # 1. 程序的顺序性规则

    规则内容：在一个线程里，按照程序编写的顺序，前面的操作Happens-Before于后续的任意操作 例子：

    class Test {
	int x = 0;
	volatile boolean v = false;
	public void writer() {
		x = 42;
		v = true;
	}
	public void reader() {
		if(v == true) {
			System.out.println(x);
		}
	}
}

    # 2. volatile 变量规则

    规则内容：对一个volatile变量的写操作Happens-Before于后续对这个volatile变量的读操作

    # 3. 传递性

    规则内容：如果 A Happens-Before B，B Happens-Before C，那么 A Happens-Before C 将规则 2 和规则 3 关联起来理解，例子如下：

    class Test {
	int x = 0;
	volatile boolean v = false;
	public void writer() {
		x = 42;
		v = true;
	}
	public void reader() {
		if(v == true) {
			System.out.println(x);
		}
	}
}

    # 4. 管程中锁的规则

    管程是一种通用的同步原语，synchronized是 Java 里对管程的实现 规则内容：对一个锁的解锁Happens-Before于后续对这个锁的加锁 例子：

    synchronized(this) {
	if(this.x < 12) {
		x = 20;
	}
}

    # 5. 线程 start() 规则（启动规则）

    规则内容：主线程 A 启动子线程 B，子线程 B 能够看到主线程 A 在启动子线程 B 之前对共享变量的操作 例子：

    Thread B = new Thread(() -> {
	System.out.println(value);
});

//A修改value的值为77
value = 77;
//主线程A启动子线程B
B.start();

    # 6. 线程 join() 规则（关于线程等待的规则）

    规则内容：主线程 A 调用join()，等待子线程 B 完成后，主线程 A 能看到子线程 B 对共享变量的操作 例子：

    Thread B = new Thread(() -> {
	value = 66;
});

value = 77;
//启动线程B，B能看到此时value = 77
B.start();
B.join();
System.out.println(value);

    # 7. 线程中断规则

    规则内容：对线程interrupt()方法的调用Happens-Before于被中断线程的代码检测到中断事件的发生 可以通过Thread.interrupted()检测线程是否中断
