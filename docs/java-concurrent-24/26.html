    26 | Fork - Join：单机版的 MapReduce

    lorraine 2022-07-11

    除了并行、聚合、批量并行这三种任务模型以外，还有一种分治的任务模型。分治：把一个复杂的问题分解成多个相似的子问题，然后再把子问题分解成更小的子问题，直到子问题可以简单到直接求解。在大数据领域中，计算机框架MapReduce背后的思想也是分治。在 Java 并发包中，提供了一种Fork / Join的并行计算框架来支持分治这种任务模型

    # 分治任务模型

    分为两个阶段：

    1. 任务分解：将任务迭代地分解成相似的子任务，直到子任务可以直接计算结果
    2. 结果合并：逐层合并子任务的执行结果，直到得到最终结果

    由于子任务是相似的，所以一般采用递归算法来解决

    # Fork / Join 框架的使用

    这个框架中 Fork 对应的是任务分解，Join 对应的是结果合并 Fork / Join框架主要包含两部分：

    1. 线程池ForkJoinPool
    2. 分治任务ForkJoinTask

    两者之间的关系就是把分治任务添加到线程池中

    # ForkJoinTask

    1. ForkJoinTask是一个抽象类，它实现了很多方法，最核心的是fork()和join()
      1. fork()：异步地执行一个子任务（任务分解）
      2. join()：阻塞当前线程来等待子任务的执行结果（结果合并）
    2. ForkJoinTask有两个子类：RecursiveAction、RecursiveTask，都是通过递归来执行任务
      1. 不同点：两者都实现了compute()方法，但是RecursiveAction定义的compute()没有返回值，RecursiveTask定义的compute()有返回值
      2. 两者都是抽象类，所以使用的时候仍然要自己定义子类来扩展

    # ForkJoinPool 的工作原理

    与ThreadPoolExecutor相比，ForkJoinPool本质上同样是“生产者 - 消费者”模式，但是不同的是ThreadPoolExecutor内部只有一个任务队列，ForkJoinPool内部有多个任务队列

    1. 线程池中的每个工作线程对应一个任务队列，这样在取任务的时候不会发生数据竞争
    2. 任务队列采用双端队列来实现

    # 提交任务

    通过ForkJoinPool的invoke()或者submit()提交任务后，会根据一定的路由规则将任务加入到一个任务队列中。如果任务执行的过程中创建了子任务，那么会将子任务提交到工作线程对应的任务队列中

    # 任务窃取

    如果工作线程对应的任务队列空了，那么就会“窃取”其他工作任务队列里的任务，这样工作线程才不会闲下来。任务队列采用的是双端队列，工作线程“正常获取任务”和“窃取任务”分别是从任务队列不同的端消费，这样能避免很多不必要的数据竞争

    # 实践

    # 1. 实现斐波那契数列

    static void main(String[] args){
	// 创建分治任务线程池
	ForkJoinPool fjp = new ForkJoinPool(4);
	// 创建分治任务
	Fibonacci fib = new Fibonacci(30); 
	// 启动分治任务
	Integer result = fjp.invoke(fib);
	// 输出结果
	System.out.println(result);
}

// 递归任务
static class Fibonacci extends RecursiveTask<Integer>{
	final int n;
	Fibonacci(int n) {
		this.n = n;
	}
	protected Integer compute(){
		if (n <= 1)
			return n;
		Fibonacci f1 = new Fibonacci(n - 1);
		// 创建子任务
		f1.fork();
		Fibonacci f2 = new Fibonacci(n - 2);
		// 等待子任务结果，并合并结果
		return f2.compute() + f1.join();
	}
}

    # 2. 模拟 MapReduce 统计单词数量

    public class CntWords {
    public static void main(String[] args) {
        String[] fc = {"hello world",
                "hello me",
                "hello fork",
                "hello join",
                "fork join in world"};
        // 创建 ForkJoin 线程池
        ForkJoinPool fjp = new ForkJoinPool(3);
        // 创建任务
        MR mr = new MR(fc, 0, fc.length);
        // 启动任务
        Map<String, Long> res = fjp.invoke(mr);
        // 输出结果
        res.forEach((k, v)-> System.out.println(k + ":" + v));
    }
}

public class MR extends RecursiveTask<Map<String, Long>> {

    private String[] fc;
    private int start, end;

    public MR(String[] fc, int start, int end) {
        this.fc = fc;
        this.start = start;
        this.end = end;
    }

    @Override
    protected Map<String, Long> compute() {
        if(start == end - 1) {
            return calc(fc[start]);
        } else {
            int mid = (start + end) / 2;
            MR mr1 = new MR(fc, start, mid);
            mr1.fork();
            MR mr2 = new MR(fc, mid, end);
            // 计算子任务，并返回合并的结果
            return merge(mr2.compute(), mr1.join());
        }
    }

    //统计单词数量
    private Map<String, Long> calc(String line) {
        Map<String, Long> map = new HashMap<>();
        // 分割单词
        String[] words = line.split("\\s+");
        // 统计单词数量
        for (String w : words) {
            Long v = map.get(w);
            if (v != null)
                map.put(w, v+1);
            else
                map.put(w, 1L);
        }
        return map;
    }

    // 合并结果
    private Map<String, Long> merge(Map<String, Long> r1, Map<String, Long> r2) {
        Map<String, Long> res = new HashMap<>();
        res.putAll(r1);
        // 合并结果
        r2.forEach((k, v) -> {
            Long c = res.get(k);
            if (c != null)
                res.put(k, c + v);
            else
                res.put(k, v);
        });
        return res;
    }
}

    # 总结

    1. 适用场景
      1. 简单的并行任务可以使用线程池 + Future
      2. 任务之间有聚合关系可以使用CompletableFuture
      3. 批量的并行任务可以使用CompletionService
      4. 分治任务模型可以使用Fork / Join框架
    2. Fork / Join并行计算框架的核心组件是ForkJoinPool。ForkJoinPool支持任务窃取机制，能够让所有线程的工作量基本均衡，不会出现有的线程很忙，而有的线程很闲的状况，所以性能很好。Java 1.8 提供的 Stream API 里面并行流也是以ForkJoinPool为基础的
    3. 默认情况下所有的并行流计算都共享一个ForkJoinPool，这个共享的ForkJoinPool默认的线程数是 CPU 的核数
    4. 如果所有的并行流计算都是 CPU 密集型计算的话，完全没有问题，但是如果存在 I / O 密集型的并行流计算，那么很可能会因为一个很慢的 I/O 计算而拖慢整个系统的性能。所以建议用不同的ForkJoinPool执行不同类型的计算任务
