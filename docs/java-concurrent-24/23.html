    23 | Future

    lorraine 2022-07-11

    在之前的例子中，ThreadPoolExecutor调用void execute(Runnable command)来提交任务，但由于这个方法没有返回值，所以无法获取任务的执行结果，但是很多场景下，是需要知道任务的执行结果的

    # 如何获取任务的执行结果

    Java 通过ThreadPoolExecutor提供的 3 个submit()方法和 1 个FutureTask 工具类来支持获得任务执行结果的需求

    # 3 个submit()方法

    这 3 个submit()方法的方法签名如下：

    // 1.提交 Runnable 任务
Future<?> submit(Runnable task);
// 2.提交 Callable 任务
<T> Future<T> submit(Callable<T> task);
// 3.提交 Runnable 任务及结果引用
<T> Future<T> submit(Runnable task, T result);

    观察上面的方法，会发现它们的返回值都是Future接口，Future接口有 5 个方法。通过 Future 接口的这 5 个方法，提交的任务不但能够获取任务执行结果，还可以取消任务。不过需要注意的是：这两个get()方法都是阻塞式的，如果被调用的时候，任务还没有执行完，那么调用get()方法的线程会阻塞，直到任务执行完才会被唤醒

    // 取消任务
boolean cancel(boolean mayInterruptIfRunning);
// 判断任务是否已取消
boolean isCancelled();
// 判断任务是否已结束
boolean isDone();
// 获得任务执行结果
get();
// 获得任务执行结果，支持超时
get(long timeout, TimeUnit unit);

    再回到 3 个submit()方法

    1. 提交Runnable任务的submit(Runnable task)：这个方法的参数是一个Runnable接口，Runnable接口的run()方法是没有返回值的，所以这个方法返回的Future仅可以用来断言任务已经结束了，类似于Thread.join()
    2. 提交Callable任务的submit(Callable<T> task)：这个方法的参数是一个Callable接口，它只有一个call()方法，并且这个方法是有返回值的，所以可以通过调用方法返回的Future对象的get()方法来获取任务的执行结果
    3. 提交Runnable任务及结果引用submit(Runnable task, T result)：假设这个方法返回的Future对象是 f，f.get()的返回值就是传给submit()方法的参数T result

    使用第三个方法的的时候，可以创建Runnable接口的实现类Task，并声明了一个有参构造函数Task(Result r)，这样创建Task对象的时候传入result，就能在类Task的run()方法中对result进行各种操作了。resul相当于主线程和子线程之间的桥梁，通过它主子线程可以共享数据

    ExecutorService executor = Executors.newFixedThreadPool(1);
// 创建 Result 对象 r
Result r = new Result();
r.setAAA(a);
// 提交任务
Future<Result> future = executor.submit(new Task(r), r); 
Result fr = future.get();
// 下面等式成立
fr === r;
fr.getAAA() === a;
fr.getXXX() === x

class Task implements Runnable{
	Result r;
	// 通过构造函数传入 result
	Task(Result r){
		this.r = r;
	}
	void run() {
		// 可以操作 result
		a = r.getAAA();
		r.setXXX(x);
	}
}

    # FutureTask 工具类

    Future是一个接口，但FutureTask是一个工具类，FutureTask有两个构造函数，与submit()类似

    FutureTask(Callable<V> callable);
FutureTask(Runnable runnable, V result);
    1
    2

    FutureTask类既实现了Runnable接口，又实现了Future接口。所以

    1. FutureTask对象可以作为任务提交给ThreadPoolExecutor去执行，也可以直接被Thread 执行
    2. FutureTask对象能用来获得任务的执行结果
    //1.作为任务提交给线程池
// 创建 FutureTask
FutureTask<Integer> futureTask = new FutureTask<>(()-> 1+2);
// 创建线程池
ExecutorService es = Executors.newCachedThreadPool();
// 提交 FutureTask 
es.submit(futureTask);
// 获取计算结果
Integer result = futureTask.get();


//2.直接被Thread执行
// 创建 FutureTask
FutureTask<Integer> futureTask
 = new FutureTask<>(()-> 1+2);
// 创建并启动线程
Thread T1 = new Thread(futureTask);
T1.start();
// 获取计算结果
Integer result = futureTask.get();

    # 用 Future 实现“烧水泡茶”程序

    “烧水泡茶”程序中，有些步骤需要串行处理，但有些步骤可以并行处理。步骤如下：

    洗水壶（1 min）-> 烧开水（15 min）---------------------> 泡茶
												   ^
												   |
洗茶壶（1 min）-> 洗茶杯（2 min）-> 拿茶叶（1 min）--+
    1
    2
    3
    4

    那么可以由两个线程来执行，线程 1 执行洗水壶、烧开水、泡茶的工作，线程 2 执行洗茶壶、洗茶杯、拿茶叶的工作，但是线程 1 在泡茶前必须要等待线程 2 执行完毕，并且线程 2 要返回给线程 1 拿的茶叶的品种 具体实现：

    // 创建任务 T2 的 FutureTask
FutureTask<String> ft2 = new FutureTask(new T2Task());
// 创建任务 T1 的 FutureTask，传入 ft2
FutureTask<String> ft1 = new FutureTask(new T1Task(ft2));
// 线程 T1 执行任务 ft1
Thread T1 = new Thread(ft1);
T1.start();
// 线程 T2 执行任务 ft2
Thread T2 = new Thread(ft2);
T2.start();
// 等待线程 T1 执行结果
System.out.println(ft1.get());


// T1Task 需要执行的任务：
// 洗水壶、烧开水、泡茶
class T1Task implements Callable<String>{
 FutureTask<String> ft2;
 // T1 任务需要 T2 任务的 FutureTask
 T1Task(FutureTask<String> ft2){
 this.ft2 = ft2;
}

// T1Task 需要执行的任务：洗水壶、烧开水、泡茶
class T1Task implements Callable<String>{
	FutureTask<String> ft2;
	
	// T1 任务需要 T2 任务的 FutureTask
	T1Task(FutureTask<String> ft2){
		this.ft2 = ft2;
	}
	
	@Override
	String call() throws Exception {
		System.out.println("T1: 洗水壶...");
		TimeUnit.SECONDS.sleep(1);
		System.out.println("T1: 烧开水...");
		TimeUnit.SECONDS.sleep(15);
		// 获取 T2 线程的茶叶
		String tf = ft2.get();
		System.out.println("T1: 拿到茶叶:"+tf);
		System.out.println("T1: 泡茶...");
		return " 上茶:" + tf;
	}
}

// T2Task 需要执行的任务：洗茶壶、洗茶杯、拿茶叶
class T2Task implements Callable<String> {
	@Override
	String call() throws Exception {
		System.out.println("T2: 洗茶壶...");
		TimeUnit.SECONDS.sleep(1);
		System.out.println("T2: 洗茶杯...");
		TimeUnit.SECONDS.sleep(2);
		System.out.println("T2: 拿茶叶...");
		TimeUnit.SECONDS.sleep(1);
		return " 龙井 ";
	}
}

    利用 Java 并发包提供的 Future 可以很容易获得异步任务的执行结果。利用多线程可以快速将一些串行的任务并行化，从而提高性能。如果任务之间有依赖关系，比如当前任务依赖前一个任务的执行结果，这种问题基本上都可以用 Future 来解决
